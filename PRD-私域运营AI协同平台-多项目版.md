# 私域运营 AI 协同平台 — 多项目版产品需求文档（PRD）

> **版本**：v1.0-MultiProject | **日期**：2026-07-07 | **状态**：初稿
> **核心定位**：支持多项目、多渠道、多账号、多 Agent 协同的智能销售自动化平台

---

## 一、产品概述

### 1.1 产品定位

本产品是一套 **多项目、多渠道的私域运营 AI 协同平台**，核心能力是：

- **多项目管理**：每个运营项目独立配置 AI 机器人人格、知识库、SOP 话术库、Top Sales 案例库，实现"一套系统服务多个品牌/多条业务线"
- **多渠道接入**：统一管理 **微信个人号、企业微信、WhatsApp 个人号、WhatsApp 企业号** 四大渠道的账号与好友
- **多 Agent 协同**：基于图一所示的协同体系，将销售全流程拆分为多个专业 Agent 按需调度协同，调用 LLM 进行拟人化沟通
- **全流程销售自动化**：覆盖从破冰 → 建立关系 → 需求挖掘 → 推荐产品 → 价格沟通 → 逼单促销 → 成交下单 → 物流跟进 → 售后服务的完整销售路径

### 1.2 与原单项目 PRD 的关键差异

| 维度 | 原单项目版本 | 多项目版本 |
|------|-------------|-----------|
| **组织结构** | 单一项目 | 多项目并行，数据隔离 |
| **AI 配置** | 全局唯一 AI 人格 | 每项目独立 AI 人格、Prompt、模型配置 |
| **知识体系** | 全局共享 | 每项目独立：知识库 / SOP / 案例 / 话术 |
| **账号归属** | 账号直接挂载系统 | 账号归属于具体项目 |
| **客户档案** | 全局唯一 | 按 `project_id + account_id + friend_id` 三元组定位 |
| **Agent 工作流** | 固定工作流 | 每项目可自定义 Agent 编排与 SOP 流程 |
| **权限管理** | 四角色固定权限 | 支持按项目分配成员与角色 |

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| **一平台多品牌** | 一套系统同时服务 N 个品牌/N 条业务线，每套配置完全隔离 |
| **7×24 自动销售** | 替代人工销售，全天候运行不疲劳 |
| **拟人化沟通** | 文本/图片/语音多模态，话术自然不机械 |
| **主动出击** | 基于朋友圈/Status 分析主动发起关怀和话题 |
| **知识严格约束** | 产品/价格/活动类回答严格基于知识库，零幻答 |
| **Agent 专业分工** | 破冰/诊断/成交/售后各有专精 Agent，总控统一合成回复 |
| **标签自动化** | 后台 4 个 Agent 并行维护剔除拉黑/进群状态/客户偏好/客户信息四类标签 |
| **群聊流量放大** | 一个群 500 人，100 个群 = 5 万触达，自动识别意向转私聊 |
| **人工无缝兜底** | 特殊场景一键转人工，处理完交还 AI |

### 1.4 目标用户

- **多品牌代理运营公司**：同时代运营多个品牌的私域
- **集团型企业**：旗下有多条业务线/子品牌，各自独立运营
- **社交电商团队**：管理大量微信号进行规模化私域销售
- **出海企业**：需要同时运营国内微信和国外 WhatsApp 渠道

### 2.2 问答接待链路 — 各模块职责

#### 2.2.1 需求分析（入口路由）

| 输入类型 | 判定逻辑 | 下游路由 |
|----------|---------|---------|
| **闲聊** | 不涉及产品/价格/购买意图 | → 闲聊处理分支（通用知识或无知识兜底） |
| **缩问问题** | 明确的产品咨询/价格询问/使用疑问 | → 知识检索模块（严格约束回答） |
| **其他课程/SOP** | 触发预定义的销售阶段推进 | → 总结模块中对应阶段专家 |
| **通用知识** | 行业常识/天气/节日等非销售话题 | → 通用知识检索 |
| **无需需求** | 纯情感表达/无实质信息 | → 直接进入表达控制生成关怀回复 |

#### 2.2.2 需求调度模块

根据需求分析结果，决定：
- 是否需要检索知识库（以及检索哪个知识库）
- 是否需要调用专业 Agent
- 是否走 SOP 预设流程
- 是否需要特殊替换（如插入客户昵称/订单信息等动态变量）

#### 2.2.3 知识检索模块

| 子模块 | 功能 | 数据源 |
|--------|------|--------|
| **知识目录检索** | 按 `project_id` 隔离的知识库分类树中检索 | 项目专属知识库（产品/价格/活动/企业介绍） |
| **QA 检索** | 精确匹配常见问答对 | 项目 QA 对库 |
| **其他辅助** | 向量相似度检索（RAG） | 项目向量化知识库 |

#### 2.2.4 新聊天打标模块

每次对话结束后自动为客户打上标签：
- **无知识标签**：该问题在知识库中未命中 → 记录为待补全知识点
- **运营标签**：标记客户的运营阶段（破冰/建关系/挖需求/引导下单/交付/售后）
- **特殊需求标签**：记录客户的特殊需求/偏好/禁忌

#### 2.2.5 总结模块（多专家汇总）

| 专家角色 | 职责 | 输出 |
|----------|------|------|
| **问聊专家** | 处理闲聊类消息，维持关系温度 | 关怀性回复草稿 |
| **缩问专家** | 处理明确的产品/价格/功能提问 | 基于知识的准确回答草稿 |
| **其他课程专家** | 处理 SOP 预设流程中的特定环节 | 符合当前销售阶段的策略建议 |
| **选用沟通专家** | 决定最佳沟通方式（文字/图片/语音/视频/文件） | 推荐的消息形式 |
| **需求沟通专家** | 综合判断下一步销售动作 | 推进建议（如：推荐产品 A、发优惠券等） |

#### 2.2.6 表达控制模块

| 能力 | 说明 |
|------|------|
| **语气控制** | 根据 project 的 AI 人格配置调整语气（专业型/亲切型/幽默型/国际化） |
| **长度控制** | 单条消息不超过配置上限（默认 200 字） |
| **表情适配** | 根据平台自动选择合适的 emoji 风格 |
| **敏感词过滤** | 过滤违规内容后再输出 |
| **去重检查** | 避免短时间内重复相同内容 |

#### 2.2.7 后置控制模块

| 能力 | 说明 |
|------|------|
| **后打断检测** | 如果客户在新消息中表达了新的紧急意图，中断当前编排流程重新路由 |
| **无知识补充** | 当知识库无法回答时，生成优雅的"我帮您确认一下再回复您"式过渡语，并创建知识缺口工单 |

#### 2.2.8 多模态发送模块

根据表达控制模块的决策，输出最终消息：
- **文本**：纯文字消息
- **图片**：从项目产品图库中选择
- **语音**：TTS 合成 或 从语音采样库选取
- **视频**：发送产品演示视频
- **文件**：发送产品手册/PDF 等
- **链接**：发送支付链接/H5 页面

### 2.3 标签自动化 — 下行 4 Agent

| Agent | 运行时机 | 维护的标签类别 | 具体行为 |
|-------|---------|---------------|---------|
| **剔除拉黑识别 agent** | 每次收到消息后异步执行 | 剔除/拉黑 | 检测辱骂/骚扰/竞品推广等行为，自动标记拉黑或降级处理 |
| **加群状态更新 agent** | 群事件触发时执行 | 进群/退群/活跃/沉睡 | 监控客户在各群的加入退出和活跃度变化 |
| **客户偏好识别 agent** | 对话结束后异步分析 | 偏好标签（兴趣品类/沟通时间/内容偏好） | 从对话历史中提取客户偏好特征 |
| **客户信息提取 agent** | 新好友添加/对话中提及信息时 | 客户基础信息（地区/行业/生日/消费能力） | 从对话中结构化提取客户画像字段 |

### 2.4 SOP 自动化

标签驱动 SOP 执行：
1. **筛选规则引擎**：基于标签组合 + 客户属性 + API 条件，精准定位目标客户群体
2. **发送规则引擎**：确定发送渠道、发送时间、发送频率、内容模板
3. **内容格式化**：支持文字/语音/视频/链接/拉群邀请/纯文本等多种格式
4. **效果追踪**：记录每次 SOP 执行的触达率、打开率、回复率、转化率


## 六、核心业务流程

### 6.1 新好友加入 → 全流程销售

```
新好友添加（通过扫码/群转/搜索/主动触达等任一途径）
    ↓
[自动] 创建 Friend 记录（project_id + account_id + friend_id）
[自动] 调用"客户信息提取 agent"提取基础信息
[自动] 设置 sales_stage = ice_breaking
[自动] 打标签：source=xxx, auto_tags.initial=true
    ↓
[破冰阶段] 发送欢迎语（项目配置的开场白模板，LLM 润色）
    ↓
客户首次回复
    ↓
[需求分析] 识别意图 → 路由到对应 Agent
    ↓
  ├── 闲聊 → 问聊专家生成关怀回复
  ├── 产品询价 → 知识检索 + 缩问专家生成准确回复
  ├── 表达需求 → 需求沟通专家引导深入
  └── 其他 → 通用知识匹配或无知识兜底
    ↓
[表达控制] 语气/长度/表情/敏感词/去重检查
    ↓
[后置控制] 检查是否需要打断或补充
    ↓
[多模态发送] 选择最佳形式发送
    ↓
[新聊天打标] 更新客户标签和销售阶段
    ↓
[标签自动化 - 异步] 4个后台 Agent 并行处理：
    ├── 剔除拉黑识别 agent
    ├── 加群状态更新 agent
    ├── 客户偏好识别 agent
    └── 客户信息提取 agent
    ↓
循环：等待下一轮客户消息...
    ↓
当客户进入引导下单阶段：
    ├── 推荐匹配产品（基于需求标签 + 产品知识库）
    ├── 告知价格（严格来自价格体系库，禁止幻答）
    ├── 处理异议（基于拒绝处理话术 + Top Sales 案例）
    └── 营造紧迫感（限时优惠/库存紧张/赠品等）
    ↓
客户下单 → 生成订单 → 推送支付链接
    ↓
支付成功 → 进入物流交付阶段
    ↓
发货 → 物流跟踪（对接快递 API）→ 签收通知
    ↓
签收确认 → 进入售后服务阶段
    ↓
使用指导 → 满意度回访 → 复购引导 → 转介绍请求
    ↓
（回到关系维护循环，持续经营）
```

### 6.2 朋友圈 → 主动私聊 → 持续对话闭环

```
手机端定时任务触发（每日 N 次浏览朋友圈）
    ↓
采集好友朋友圈/Status 内容（文本+图片）
    ↓
上传服务器 → 存入 moment 表
    ↓
服务器端 LLM 语义分析：
  ├── mood: 近况情绪
  ├── intent_signal: 商机信号强度
  ├── business_relevance: 业务相关度
  ├── risk_level: 风险等级
  ├── recommended_opening_style: 建议开场方式
  └── do_not_say: 禁忌话题
    ↓
高风险/负面内容 → 跳过，仅记录
低质/无意义内容 → 降低频次
有价值内容 → 继续
    ↓
[可选] 朋友圈点赞（手机端执行，仅点赞不评论）
    ↓
判断是否有主动触达机会：
  ✅ mood 积极 AND business_relevance 高/中 AND 24h内未触达过
  ❌ 其他情况跳过
    ↓
AI 生成首条私聊消息：
  ├── internal_header（内部日志用，不发送给客户）
  ├── plain_text（实际发送的自然语言内容）
  ├── search_aliases（手机端搜索用的别名列表）
  ├── verify_fields（二次验证字段）
  └── policy（频控策略参数）
    ↓
频控审核通过
    ↓
下发 proactive_reach 指令到手机端（MQTT）
    ↓
手机端执行：
  打开社交App → 用search_aliases搜索 → 点候选 → 进资料页二次验证
  → 通过则进入聊天窗口 → 仅发送 plain_text → 记录conversation_id
    ↓
监听好友回复
    ↓
好友回复 → 进入 AI 持续对话模式（conversation_reply 循环）
    ↓
记录主动触达日志 + 更新会话状态 + 更新最后联系时间
```

### 6.3 群聊 → 意趣识别 → 转私聊 → 成交

```
群消息到达
    ↓
[兴趣识别]
  强信号（怎么买/多少钱/下单）→ score += 10
  中信号（怎么样/好用吗/区别）→ score += 5
  弱信号（点赞/长阅读）→ score += 2
  互动频次加分 + 活跃天数加分
    ↓
总分达到阈值（默认30分）？
  ├── 否 → 保持观察，继续积累分数
  └── 是 → 触发转出
    ↓
[自动转出]
  方式A: 直接私聊："您好，我是XX群的助理..."
  方式B: 群内@用户："这个问题我单独给您详细介绍～"
  方式C: 推送名片/二维码引导主动添加
    ↓
转入一对一私聊销售流程（见 6.1）
    ↓
标记"已转出"，群内不再重复触发
    ↓
成交后回写：标记该客户在群内为"已转化"
```

## 七、移动端能力（手机客户端）

### 7.1 核心页面

| 页面 | 功能 |
|------|------|
| **首页** | 设备状态、MQTT 连接、关联账号数、今日统计、AI 模式、待确认动作 |
| **账号** | 按平台展示各账号在线状态/好友数/群数/同步状态 |
| **待处理** | 人工确认的动作任务列表（消息发送/点赞/定时发送/异常处理） |
| **动态** | 朋友圈/Status 采集流 + AI 建议入口 |
| **社交** | 好友/群组列表 + 同步按钮 + 1V1/群聊会话查看 |
| **绑定** | 扫码绑定 / 手动输入设备码 + 绑定状态展示 |
| **日志** | 系统操作日志（按类型/结果/时间筛选） |
| **设置** | 服务器地址、屏幕常亮、同步频率、动作确认模式 |

### 7.2 平台差异化行为

| 操作 | 微信个人号 | 企业微信 | WhatsApp 个人号 | WhatsApp 企业号 |
|------|-----------|---------|---------------|---------------|
| 消息监听 | AccessibilityService | API Webhook | AccessibilityService | API Webhook |
| 消息发送 | 无障碍模拟 | API 发送 | 无障碍模拟 | API 发送 |
| 联系人同步 | 无障碍遍历 | API 获取 | 无障碍遍历 | API 获取 |
| 社交动态采集 | 朋友圈浏览 | 手机端辅助 | Status 浏览 | 手机端辅助 |
| 社交互动 | 点赞 | 点赞 | 互动 | 互动 |
| 防封策略 | 严格 | 无需 | 中等 | 无需 |

### 7.3 执行模式

每种动作支持三种执行模式：
- **auto**：全自动执行，无需人工干预
- **manual_confirm**：生成待确认任务，手机端展示等待用户确认
- **disabled**：禁用该动作

## 九、术语表
