# 接口设计文档 - 无代码编排工作流（面向 Morphix 集成与复用）

> **版本**：v1.0 | **日期**：2026-07-11 | **状态**：初稿
> **目标**：让无代码编排子系统能够被正在开发的 Morphix 主系统直接整合调用，或以可复用能力中心的方式对外提供服务。

---

## 一、文档目标

本文档定义无代码编排工作流子系统对 Morphix 主系统开放的接口边界、调用方式、鉴权规则、事件机制、错误码和集成建议，重点解决两类问题：

1. **整合调用**：Morphix 主系统如何嵌入编排、调试、发布、运行和追踪能力。
2. **能力复用**：Morphix 现有机器人、知识库、素材库、标签、会话与渠道能力如何被编排系统复用，避免重复建设。

---

## 二、集成定位

### 2.1 集成原则

| 原则 | 说明 |
|------|------|
| 主系统统一入口 | Morphix 仍作为统一管理后台和业务入口，无代码编排作为核心能力子系统接入 |
| 定义与运行解耦 | 工作流定义接口、发布接口、运行接口分别建模，避免编辑态与生产态耦合 |
| 项目上下文强约束 | 所有核心接口必须显式携带 `project_id`，确保多项目隔离 |
| 机器人为主要挂载点 | 工作流优先绑定到 `bot_id`，供 Morphix 机器人直接调用 |
| 渠道能力复用而非重造 | 企微、微信个人号、WhatsApp、Morphix 标签等能力优先复用主系统现有服务 |
| 发布态对外运行 | 生产会话默认只调用已发布版本；草稿态仅允许调试接口调用 |
| 事件驱动补齐闭环 | 发布成功、回滚成功、运行完成、运行失败等状态通过事件通知 Morphix |

### 2.2 在 Morphix 中的角色定位

| 系统/模块 | 定位 |
|------|------|
| Morphix 主系统 | 统一门户、项目管理、机器人管理、会话管理、客户管理、权限主体 |
| 无代码编排子系统 | 负责流程定义、校验、调试、发布、执行和运行追踪 |
| 知识库/素材库服务 | 作为可复用资产服务，被编排系统查询和绑定 |
| 渠道适配服务 | 复用现有渠道账号、标签、拉群、托管和消息发送能力 |
| LLM 网关 | 复用 Morphix 现有模型路由和密钥管理能力 |
| 审计与监控平台 | 接收编排侧审计、运行日志和指标 |

## 三、推荐集成模式

### 3.1 模式 A：后台嵌入式集成（推荐）

Morphix 后台直接嵌入“编排工作台 / 对话测试台 / 版本管理台”，统一登录态和项目上下文。

**适用场景：**
- Morphix 需要在同一后台内完成机器人创建、编排、调试、发布、上线
- 需要复用现有权限体系和导航结构

**优点：**
- 用户体验统一
- 权限和项目上下文天然一致
- 复用现有机器人、知识库、素材库、渠道配置更直接

### 3.2 模式 B：运行时服务化调用（推荐）

Morphix 会话引擎、托管服务或消息网关，在收到用户消息后，调用工作流运行接口，由编排执行引擎返回输出动作。

**适用场景：**
- 机器人托管逻辑已存在，需要把“回复决策”部分逐步切换为可编排流程
- 后续需要支持定时任务、Webhook、内部事件等多触发源

**优点：**
- 与现有会话主链路兼容
- 易于灰度切换和回滚
- 便于做同步/异步运行拆分

### 3.3 模式 C：能力中心复用模式

无代码编排只提供“流程定义 + 发布版本 + 执行引擎”，知识库、素材库、标签、客户属性、渠道消息等能力继续调用 Morphix 现有域服务。

**适用场景：**
- 避免重复建设数据资产模块
- 希望后续将编排能力推广到更多 Morphix 子模块（运营 SOP、AI 机器人、客户自动化）

### 3.4 模式 D：事件协同模式

发布、回滚、上线、运行完成、运行失败等状态，以事件或回调形式通知 Morphix 其他模块。

**适用场景：**
- 需要驱动运营任务、消息日志、客户标签、监控告警等联动逻辑

## 四、总体接口边界

### 4.1 编排子系统对外提供的接口域

| 接口域 | 说明 | 调用方 |
|------|------|------|
| 工作流定义接口 | 创建、编辑、查询、复制、删除工作流 | Morphix 后台 |
| 节点与配置元数据接口 | 节点列表、Schema、分类、能力约束 | Morphix 后台 |
| 草稿校验与保存接口 | 校验流程结构、变量引用、参数完整性 | Morphix 后台 |
| 调试接口 | 创建调试会话、执行草稿、查看节点轨迹 | Morphix 后台 / 测试台 |
| 发布治理接口 | 发布、回滚、版本列表、版本差异 | Morphix 后台 |
| 运行时接口 | 执行发布版工作流、按机器人派发消息、查询运行结果 | Morphix 会话引擎 / 托管服务 |
| 追踪与日志接口 | 查询运行详情、节点执行轨迹、异常信息 | Morphix 后台 / 运维平台 |
| 集成回调接口 | 事件通知与状态同步 | Morphix 其他模块 |

### 4.2 编排子系统需要复用 Morphix 的接口域

| 复用域 | 说明 |
|------|------|
| 项目与组织 | 获取项目、成员、角色、项目状态 |
| 用户与权限 | 统一登录、角色鉴权、操作人身份 |
| 机器人管理 | 机器人基础信息、欢迎语、托管状态、上线状态 |
| 渠道账号管理 | 账号绑定、渠道在线状态、授权信息 |
| 客户与会话 | 客户 ID、会话 ID、渠道会话上下文、最近聊天记录 |
| 标签服务 | 平台标签读写、企微标签读写、分组拉群能力 |
| 知识库服务 | 知识库列表、搜索、检索配置、可见范围 |
| 素材库服务 | 图片/视频/文件素材查询与发送前资源解析 |
| LLM 网关 | 模型路由、调用限流、API Key 托管、超时和重试策略 |
| 审计与监控 | 操作留痕、指标上报、告警渠道 |

## 五、统一接口规范

### 5.1 协议规范

| 项 | 规范 |
|------|------|
| 协议 | HTTPS |
| 数据格式 | `application/json; charset=utf-8` |
| 字符编码 | UTF-8 |
| 时间格式 | ISO 8601 |
| ID 类型 | 建议统一为 `string`（可承载 uuid / snowflake） |
| 幂等 | 写操作支持 `X-Idempotency-Key` |
| 链路追踪 | 全链路透传 `X-Trace-Id` |

### 5.2 URL 规范

建议以独立前缀提供服务：

```text
/api/morphix/workflow/v1
```

示例：

```text
GET  /api/morphix/workflow/v1/workflows
POST /api/morphix/workflow/v1/workflows
POST /api/morphix/workflow/v1/runtime/bots/{botId}/dispatch
```

### 5.3 请求头规范

| Header | 必填 | 说明 |
|------|------|------|
| Authorization | 是 | 用户态 JWT 或内部服务 Token |
| X-Project-Id | 是 | 当前项目 ID |
| X-Operator-Id | 否 | 操作人 ID，便于审计 |
| X-Bot-Id | 否 | 当前机器人 ID |
| X-Trace-Id | 是 | 调用链追踪 ID |
| X-Idempotency-Key | 写接口建议 | 防重复发布、防重复触发 |
| X-Source-System | 建议 | 标识调用来源，如 `morphix-admin` / `morphix-runtime` |

### 5.4 统一响应体

```json
{
  "code": 0,
  "message": "ok",
  "data": {},
  "requestId": "trace_xxx"
}
```

说明：
- `code = 0` 表示成功
- 非 0 表示业务错误
- `requestId` 与 `X-Trace-Id` 对齐，便于跨服务排查

### 5.5 鉴权模型

| 场景 | 鉴权方式 |
|------|------|
| 后台用户操作 | 使用 Morphix 统一登录 JWT + 项目角色校验 |
| 服务间运行调用 | 使用内部服务 Token / HMAC 签名 |
| 回调通知 | 使用签名头 `X-Signature` + 时间戳防重放 |
| 高危动作（发布/回滚） | 除基础权限外，可加二次确认或审批校验 |

### 5.6 资源作用域

所有核心资源都必须校验以下作用域：

| 资源 | 作用域要求 |
|------|------|
| workflow | 必须属于 `project_id` |
| workflow_version | 必须属于 `project_id` 且隶属指定 workflow |
| bot | 必须属于 `project_id` |
| knowledge_base | 必须属于 `project_id` 或项目共享范围 |
| material_asset | 必须属于 `project_id` 或机器人共享范围 |
| debug_session / workflow_run | 必须属于 `project_id` |

## 六、核心数据映射

### 6.1 Morphix 与编排子系统对象映射

| Morphix 对象 | 编排对象 | 说明 |
|------|------|------|

| 知识库 | `knowledge_base_id` | 检索资产 |
| 素材 | `material_asset_id` | 输出资产 |

## 七、接口清单设计

## 7.1 工作流定义接口

### 7.1.1 创建工作流

**POST** `/api/morphix/workflow/v1/workflows`

请求体：

```json
{
  "botId": "bot_2001",
  "name": "销售线索首轮接待",
  "description": "用于线索进入后的首轮欢迎与意图识别",
  "workflowType": "main"
}
```

响应重点字段：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "workflowId": "wf_1001",
    "latestDraftId": "draft_1001"
  },
  "requestId": "trace_xxx"
}
```

### 7.1.2 查询工作流列表

**GET** `/api/morphix/workflow/v1/workflows?botId=bot_2001&keyword=销售&status=active`

支持条件：
- `botId`
- `workflowType`
- `status`
- `keyword`
- `pageNo`
- `pageSize`

### 7.1.3 查询工作流详情

**GET** `/api/morphix/workflow/v1/workflows/{workflowId}`

返回：
- 基础信息
- 当前草稿信息
- 当前发布版本信息
- 最近更新时间
- 绑定机器人信息

### 7.1.4 更新工作流元数据

**PUT** `/api/morphix/workflow/v1/workflows/{workflowId}`

用于修改名称、描述、类型、绑定机器人等基础信息。

### 7.1.5 删除/归档工作流

**DELETE** `/api/morphix/workflow/v1/workflows/{workflowId}`

说明：
- 默认执行逻辑归档，不建议物理删除
- 若已有发布版本，需校验是否允许归档

### 7.1.6 复制工作流 / 生成模板

**POST** `/api/morphix/workflow/v1/workflows/{workflowId}/clone`

请求体：

```json
{
  "targetProjectId": "p_1002",
  "targetBotId": "bot_2101",
  "name": "销售线索首轮接待-复制",
  "cloneMode": "template_copy"
}
```

适用于跨项目复用与快速实施。

## 7.2 节点元数据接口

### 7.2.1 获取节点分类与节点列表

**GET** `/api/morphix/workflow/v1/node-definitions`

返回内容：
- 节点分类
- 节点编码
- 节点名称
- 输入输出类型
- 参数 Schema
- 是否需要渠道授权

### 7.2.2 获取节点详情

**GET** `/api/morphix/workflow/v1/node-definitions/{nodeType}`

适用于前端节点配置面板动态渲染。

### 7.2.3 获取渠道能力约束

**GET** `/api/morphix/workflow/v1/channel-capabilities?channel=wecom`

返回：
- 支持的动作节点
- 支持的输出类型
- 授权前置条件
- 不支持能力说明

## 7.3 草稿保存与校验接口

### 7.3.1 保存草稿

**POST** `/api/morphix/workflow/v1/workflows/{workflowId}/draft:save`

请求体：

```json
{
  "definition": {
    "nodes": [],
    "edges": [],
    "layout": {},
    "globalVariables": []
  }
}
```

保存后返回：
- `draftId`
- `validationStatus`
- `validationErrors`

### 7.3.2 校验草稿

**POST** `/api/morphix/workflow/v1/workflows/{workflowId}/draft:validate`

校验内容：
- 必填输入点是否完整
- 输入输出类型是否匹配
- 变量引用是否合法
- 知识库/素材/子流程引用是否有效
- 渠道节点前置条件是否满足

响应示例：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "valid": false,
    "errors": [
      {
        "nodeKey": "node_ai_01",
        "field": "prompt",
        "errorCode": "WF_VALIDATION_REQUIRED",
        "message": "AI 对话节点提示词不能为空"
      }
    ]
  },
  "requestId": "trace_xxx"
}
```

## 7.4 调试接口

### 7.4.1 创建调试会话

**POST** `/api/morphix/workflow/v1/workflows/{workflowId}/debug-sessions`

请求体：

```json
{
  "runMode": "draft",
  "draftId": "draft_1002",
  "inputType": "text",
  "inputPayload": {
    "text": "你好，帮我介绍一下课程"
  },
  "context": {
    "customerId": "c_4001",
    "conversationId": "cv_5001",
    "channel": "wecom"
  }
}
```

说明：
- `runMode` 支持 `draft` / `published_version`
- 调试时允许加载草稿

### 7.4.2 获取调试详情

**GET** `/api/morphix/workflow/v1/debug-sessions/{debugSessionId}`

返回内容：
- 最终输出
- 节点执行顺序
- 每个节点输入/输出
- 错误信息
- 总耗时

### 7.4.3 清空调试上下文

**POST** `/api/morphix/workflow/v1/debug-sessions/{debugSessionId}/clear-context`

适用于重新调试时重置上下文记忆。

## 7.5 发布治理接口

### 7.5.1 发布工作流

**POST** `/api/morphix/workflow/v1/workflows/{workflowId}/publish`

请求体：

```json
{
  "draftId": "draft_1002",
  "versionName": "v1.0.0",
  "changeSummary": "新增知识库检索与标签打标逻辑",
  "targetEnv": "prod"
}
```

执行动作：
1. 校验发布权限
2. 校验草稿结构与外部依赖
3. 生成不可变版本快照
4. 更新机器人当前生效版本
5. 写入审计日志
6. 发送发布成功事件

### 7.5.2 查询版本列表

**GET** `/api/morphix/workflow/v1/workflows/{workflowId}/versions`

### 7.5.3 获取版本详情

**GET** `/api/morphix/workflow/v1/versions/{versionId}`

### 7.5.4 版本差异对比

**GET** `/api/morphix/workflow/v1/versions/compare?from=v_1001&to=v_1002`

返回：
- 节点增删改
- 连线变化
- 参数差异
- 依赖差异（知识库、子流程、渠道节点）

### 7.5.5 回滚版本

**POST** `/api/morphix/workflow/v1/workflows/{workflowId}/rollback`

请求体：

```json
{
  "targetVersionId": "v_1001",
  "reason": "新版本触发链路异常，回滚到稳定版本"
}
```

## 7.6 运行时接口（Morphix 主系统重点对接）

### 7.6.1 按机器人派发会话事件（推荐主入口）

**POST** `/api/morphix/workflow/v1/runtime/bots/{botId}/dispatch`

> 推荐作为 Morphix 会话主链路调用入口。Morphix 收到消息后，按机器人上下文把事件派发给工作流引擎。

请求体：

```json
{
  "projectId": "p_1001",
  "triggerType": "user_message",
  "channel": "wecom",
  "channelAccountId": "ca_3001",
  "customerId": "c_4001",
  "conversationId": "cv_5001",
  "message": {
    "messageId": "msg_6001",
    "type": "text",
    "text": "我想了解价格"
  },
  "runtimeOptions": {
    "timeoutMs": 8000,
    "returnTrace": false
  }
}
```

响应体建议：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "runId": "run_7001",
    "workflowId": "wf_1001",
    "workflowVersionId": "v_1002",
    "status": "success",
    "outputs": [
      {
        "type": "message",
        "content": "您好，我来为您介绍一下套餐。"
      },
      {
        "type": "set_morphix_tags",
        "tags": ["价格咨询"]
      }
    ]
  },
  "requestId": "trace_xxx"
}
```

### 7.6.2 直接执行指定工作流版本

**POST** `/api/morphix/workflow/v1/runtime/workflows/{workflowId}/execute`

适用于：
- 内部运营任务调用
- 定时触发调用
- Webhook 触发调用
- 指定版本灰度验证

### 7.6.3 查询运行结果

**GET** `/api/morphix/workflow/v1/runtime/runs/{runId}`

返回：
- 最终状态
- 输出动作
- 总耗时
- 版本号
- 错误摘要

### 7.6.4 查询节点运行轨迹

**GET** `/api/morphix/workflow/v1/runtime/runs/{runId}/nodes`

适用于运维排查和会话回放。

### 7.6.5 重放运行

**POST** `/api/morphix/workflow/v1/runtime/runs/{runId}/replay`

适用于：
- 问题复现
- 回归验证
- 新旧版本对比

## 7.7 资源复用接口

### 7.7.1 获取可用知识库列表

**GET** `/api/morphix/workflow/v1/integration/knowledge-bases?botId=bot_2001`

说明：
- 实际数据可透传或聚合自 Morphix 知识库服务

## 八、回调与事件设计

## 8.1 编排侧对 Morphix 的回调事件

| 事件名 | 触发时机 | 主要用途 |
|------|------|------|
| `workflow.published` | 工作流发布成功 | 更新机器人当前生效版本、触发上线记录 |
| `workflow.rolled_back` | 工作流回滚成功 | 恢复稳定版本、写运行公告 |
| `workflow.run.finished` | 一次运行完成 | 写消息日志、更新会话结果 |
| `workflow.run.failed` | 一次运行失败 | 告警、故障统计、兜底处理 |
| `workflow.debug.finished` | 调试结束 | 前端展示调试完成状态 |
| `workflow.dependency.invalid` | 引用资产失效 | 提醒知识库/素材/子流程依赖异常 |

### 8.2 回调接口建议

**POST** `/api/morphix/internal/workflow/events`

事件示例：

```json
{
  "eventType": "workflow.published",
  "eventId": "evt_9001",
  "occurredAt": "2026-07-11T10:30:00+08:00",
  "projectId": "p_1001",
  "botId": "bot_2001",
  "workflowId": "wf_1001",
  "workflowVersionId": "v_1002",
  "operatorUserId": "u_101",
  "payload": {
    "versionName": "v1.0.0",
    "changeSummary": "新增知识路由和打标逻辑"
  },
  "signature": "sha256=xxx"
}
```

### 8.3 Morphix 对编排侧的事件输入

建议由 Morphix 主系统向编排侧发送以下触发事件：

| 事件名 | 说明 |
|------|------|
| `conversation.message.received` | 收到用户消息 |
| `conversation.timeout.reached` | 达到超时节点需要的等待/延迟触发 |
| `bot.hosting.enabled` | 机器人托管已开启 |
| `bot.hosting.disabled` | 机器人托管已关闭 |
| `customer.tag.changed` | 客户标签变更，可触发后续流程 |
| `internal.schedule.triggered` | 定时任务触发工作流 |

## 九、错误码设计

| 错误码 | 含义 | 说明 |
|------|------|------|
| `WF_NOT_FOUND` | 工作流不存在 | ID 错误或项目不匹配 |
| `WF_DRAFT_NOT_FOUND` | 草稿不存在 | 草稿已删除或不可访问 |
| `WF_VERSION_NOT_FOUND` | 版本不存在 | 回滚/查看时版本错误 |
| `WF_PERMISSION_DENIED` | 无权限 | 用户角色无查看/编辑/发布权限 |
| `WF_VALIDATION_REQUIRED` | 必填项缺失 | 节点参数不完整 |
| `WF_VALIDATION_TYPE_MISMATCH` | 类型不匹配 | 节点输入输出连线非法 |
| `WF_VARIABLE_INVALID` | 变量引用错误 | 变量不存在或作用域错误 |
| `WF_DEPENDENCY_INVALID` | 外部依赖无效 | 知识库/素材/子流程/渠道授权异常 |
| `WF_PUBLISH_BLOCKED` | 不允许发布 | 校验未通过或存在阻断依赖 |
| `WF_ROLLBACK_BLOCKED` | 不允许回滚 | 目标版本不可恢复 |
| `WF_RUNTIME_TIMEOUT` | 运行超时 | 节点或整链路执行超时 |
| `WF_RUNTIME_FAILED` | 运行失败 | 执行器异常 |
| `WF_CHANNEL_UNSUPPORTED` | 渠道不支持 | 当前渠道不支持某节点能力 |
| `WF_CHANNEL_AUTH_REQUIRED` | 渠道授权缺失 | 企微、托管等前置条件未满足 |
| `WF_KB_NOT_READY` | 知识库未就绪 | 知识索引未完成 |
| `WF_ASSET_NOT_FOUND` | 素材不存在 | 素材被删除或无权限 |

## 十、接口实现建议（结合当前 Morphix 开发）

### 10.1 推荐实施形态

结合 Morphix 当前在研状态，建议采用：

**“后台嵌入 + 运行时服务化 + 资产复用”三段式方案**

| 层 | 建议 |
|------|------|
| 管理层 | 在 Morphix 后台直接嵌入编排工作台、调试台、版本页 |
| 服务层 | 把无代码编排做成独立服务域，如 `workflow-service` |
| 运行层 | 由 Morphix 会话网关调用 `runtime dispatch` 接口 |
| 资产层 | 知识库、素材库、标签、客户属性继续复用主系统域服务 |
| 监控层 | 编排运行日志接入 Morphix 统一监控和审计平台 |

### 10.2 推荐服务拆分

| 服务 | 职责 |
|------|------|
| workflow-definition-service | 工作流定义、草稿保存、节点元数据、变量校验 |
| workflow-release-service | 发布、回滚、版本管理、审计 |
| workflow-runtime-service | 工作流执行、节点调度、运行追踪 |
| workflow-integration-gateway | 对接 Morphix 项目/机器人/知识库/素材/渠道服务 |

### 10.3 与现有 Morphix 模块的优先对接顺序

| 顺序 | 对接模块 | 原因 |
|------|------|------|
| P0 | 机器人管理 | 工作流要挂载到机器人上，形成最小业务闭环 |
| P0 | 会话/托管服务 | 运行时必须能接住真实用户消息 |
| P0 | 项目与权限 | 多项目隔离和后台访问控制必须先打通 |
| P1 | 知识库服务 | 直接支撑知识机器人和专业问答场景 |
| P1 | 素材库服务 | 支撑多模态消息输出 |
| P1 | 标签/客户属性服务 | 支撑打标、记忆、客户状态流转 |
| P2 | 审计与监控 | 面向生产稳定性和治理闭环 |

## 十二、验收标准

| 编号 | 验收项 | 标准 |
|------|------|------|
| API-01 | Morphix 后台可拉起编排工作台 | 已登录用户进入指定机器人后可打开工作流编辑界面 |
| API-02 | 工作流可绑定机器人 | 发布后机器人能切换到新版本 |
| API-03 | Morphix 会话可调用运行时接口 | 用户消息到达后，工作流可稳定返回输出动作 |
| API-04 | 草稿与发布态隔离 | 草稿仅可调试，不会影响生产运行 |
| API-05 | 版本回滚可生效 | 回滚后新消息按旧版本执行 |
| API-06 | 资产复用有效 | 知识库、素材库、标签、客户属性均可在流程中调用 |
| API-07 | 审计可追踪 | 创建、保存、发布、回滚、调试操作均有记录 |
| API-08 | 错误可定位 | 运行失败能通过 `runId` 和 `traceId` 查到节点级错误 |

## 十三、结论与建议

这套接口设计的核心不是把无代码编排做成一个孤立新系统，而是把它设计成 **Morphix 的可复用流程能力中心**。

建议你在实施时坚持三点：

1. **主系统统一入口不变**：用户仍在 Morphix 中完成机器人管理与流程操作。
2. **运行时入口统一收口**：所有生产消息优先走 `runtime/bots/{botId}/dispatch`，避免多入口失控。
3. **资产服务尽量复用**：知识库、素材库、标签、客户属性、渠道动作不要重复建设，只做编排层抽象和适配。
