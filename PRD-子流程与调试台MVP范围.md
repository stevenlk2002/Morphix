# 编排台增量 PRD：子流程打包复用 & 对话测试调试台

> **版本**：v1.0 | **日期**：2026-07-14 | **作者**：许清楚（PM）
> **前置依赖**：《PRD-编排台MVP实现范围.md》v1.0、《PRD-无代码编排工作流.md》v1.0 5.7/5.10 节
> **当前实现基线**：React Flow 画布 + 13 节点 + localStorage 持久化 + 校验/导出

---

## 一、背景与目标

编排台 MVP（11 P0）已完成，用户可在画布上搭建完整工作流。接下来需要解决两个核心瓶颈：

1. **复用问题**：成熟流程无法跨 Bot 复用，每次需手动重建 → 需子流程打包复用能力
2. **验证问题**：编排完成后无法本地验证流程正确性，只能发布后观测 → 需对话测试调试台

两个功能属增量 P0，均在当前编排台前端内实现，不依赖后端 API 改造。

## 二、功能 1：子流程打包复用

### 2.1 P0 必做项（5 项）

| 编号 | 功能 | 说明 |
|------|------|------|
| **SF-P0-01** | 多选节点 + "打包为子流程" | 画布框选/Shift 多选节点 → 工具栏出现"打包为子流程"按钮 → 点击弹出配置弹窗 |
| **SF-P0-02** | 子流程接口定义弹窗 | 弹窗填写：子流程名称（必填）、描述（选填）；系统自动分析：将子流程内所有未连线输入端口暴露为子流程输入接口、所有未连线输出端口暴露为子流程输出接口；用户可勾选/取消各接口；确认后生成子流程定义 |
| **SF-P0-03** | 子流程 localStorage 存储 | 独立 key `morphix_subflow_{id}`，存储 nodes / edges / interface（输入输出接口）/ name / version / createdAt / updatedAt；子流程节点 ID 重新生成（避免与主流程冲突） |
| **SF-P0-04** | 子流程在复合面板展示 | 复合 Tab → "子流程调用"分类下动态展示已保存的子流程（替换占位节点）；每个子流程显示名称 + 输入/输出端口数；支持拖入画布 |
| **SF-P0-05** | 子流程调用节点 + 展开查看 | 拖入画布后渲染为单节点卡片（子流程名称 + 输入/输出端口）；点击节点 → 属性面板显示"展开查看内部"按钮 → 点击后在弹窗中以只读 React Flow MiniMap 查看内部结构（不可编辑画布，仅展示节点和连线拓扑） |

### 2.2 UI 交互流程

```
画布框选 2+ 节点
  → 工具栏浮现 "打包为子流程" 按钮
  → 点击按钮 → 弹出子流程定义弹窗
      ├─ 名称（必填，如 "新客户欢迎流程"）
      ├─ 描述（选填）
      ├─ 输入接口预览（自动分析，可勾选/取消）
      │   例：userInput.output → 暴露为子流程输入 "用户输入"
      ├─ 输出接口预览（自动分析，可勾选/取消）
      │   例：msgOutput.input ← 暴露为子流程输出 "消息输出"
      └─ [确认打包] [取消]
  → 确认后：
      1. 画布上被选中节点替换为 1 个子流程调用节点
      2. 子流程数据写入 localStorage
      3. 复合面板 "子流程调用" 分类刷新显示
      4. toast "子流程已保存"

复合面板 → "子流程调用" → 拖入画布
  → 生成子流程调用节点（subflowCall 类型）
  → 点击节点 → 属性面板
      ├─ 子流程名称（只读）
      ├─ 子流程描述（只读）
      ├─ 输入接口列表（可连线/输入值）
      ├─ 输出接口列表（供其他节点连线）
      └─ [展开查看内部] 按钮
          → 弹出 Modal（800×600）→ React Flow 只读画布
              ├─ MiniMap 缩略图 + 只读节点/连线
              ├─ 不可拖拽、不可编辑
              └─ [关闭]
```

### 2.3 数据模型要点

```typescript
// 子流程接口定义
interface SubflowPortDef {
  key: string;           // 原始端口 key
  varName: string;       // 变量名
  dataType: PortDataType; // 端口类型
  direction: 'input' | 'output';
  label: string;         // 显示名称
}

// 子流程持久化格式
interface SubflowPersisted {
  id: string;                  // UUID
  name: string;
  desc: string;
  interface: {
    inputs: SubflowPortDef[];  // 暴露的输入接口
    outputs: SubflowPortDef[]; // 暴露的输出接口
  };
  nodes: SerializedNode[];     // 内部节点（ID 已重新生成）
  edges: SerializedEdge[];     // 内部连线
  createdAt: string;
  updatedAt: string;
  version: number;             // 固定为 1，MVP 不做版本管理
}

// localStorage key
// morphix_subflow_{id} → SubflowPersisted
// morphix_subflow_index → string[] （所有子流程 ID 列表，用于面板加载）
```

### 2.4 子流程调用节点的端口呈现规则

- 子流程定义中的 `interface.inputs` → 在调用节点上渲染为**输入端口**（左侧 handle）
- 子流程定义中的 `interface.outputs` → 在调用节点上渲染为**输出端口**（右侧 handle）
- 端口类型继承自原始节点端口类型（支持 7 色类型校验）
- 节点卡片标题显示子流程名称，副标题显示 "X 入 / Y 出"

### 3.3 Mock 执行策略

| 节点类型 | Mock 策略 |
|---------|----------|
| **用户输入** | 使用测试面板输入的消息文本，输出 `userChatInput` |
| **AI 对话** | 返回固定 mock 文本：`"[Mock] 这是 AI 对话节点的模拟回复。输入: {userInput}"`；systemPrompt/config 从节点配置读取但不实际调用 LLM |
| **知识库搜索** | 返回 mock 结果 JSON：`[{ "content": "[Mock] 知识库搜索结果", "score": 0.95, "kbId": "mock-kb-1" }]` |
| **多重判断器** | 按节点配置的规则做前端字符串/数值匹配，输出 `true`/`false` 分支 |
| **正则提取 / JSON 提取** | 对上游输出值执行前端正则/JSON 解析，真实执行 |
| **消息输出** | 不实际发送，将输出内容记录到执行列表（标记 `sent`） |
| **全局变量** | 直接使用节点配置的默认值 |
| **对话记录获取** | 返回 mock 历史：`[{ role: "user", content: "mock history 1" }, ...]` |
| **时间控制** | 跳过等待，直接传递输入值（不实际 sleep） |
| **设置客户属性** | 模拟写入，在执行详情中记录属性名和值 |
| **智能体嵌入** | 返回 mock 文本：`"[Mock] 智能体嵌入节点模拟输出"` |
| **子流程调用** | 按子流程内部节点拓扑逐节点 mock 执行（递归），对外暴露子流程输出接口的值 |
| **未实现节点** | 透传输入值作为输出，标记 ⚠ "节点尚未实现，数据透传" |

### 3.4 拓扑执行算法（前端 JS）

```
function mockExecute(workflow, userMessage):
  1. 构建 DAG（nodes + edges）
  2. 拓扑排序确定执行顺序
  3. 初始化 context = { userChatInput: userMessage }
  4. 按拓扑序依次执行每个节点：
     a. 收集所有上游输出 → 填入节点 inputs
     b. 按节点类型调用对应 mock 函数
     c. 将节点 outputs 写入 context
     d. 记录执行详情到 executionTrace[]
  5. 对于判断器分支：仅执行匹配的分支路径
  6. 返回 executionTrace
```

## 六、待确认问题

1. **子流程调用节点的删除行为**：从画布删除子流程调用节点时，是否同时删除子流程定义本身？建议：删除调用节点仅解除引用，子流程定义保留在 localStorage（用户可在面板中重新拖入）。

2. **子流程的输入接口自动分析边界**：若子流程内部有多个"用户输入"节点，是否全部暴露为输入接口？建议：所有无上游连线的输入端口均暴露，由用户在弹窗中手动取舍。

3. **测试面板与属性面板的共存**：测试模式下右侧属性面板是否隐藏？建议：测试模式下属性面板折叠（只读画布无需配置），测试面板占据底部，节省空间。

4. **快捷消息预设来源**：快捷消息是内置固定列表，还是从工作流中的"用户输入"节点配置读取？建议：内置 3-5 条通用消息（"你好"、"我想了解产品"、"帮我查询订单"），后续可由用户在测试面板中自定义保存。
