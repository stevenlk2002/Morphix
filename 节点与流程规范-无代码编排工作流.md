# 节点与流程规范 - 无代码编排工作流

> **版本**：v1.0 | **日期**：2026-07-11

---

## 一、文档目的

本文档定义无代码编排工作流的节点模型、连线规则、变量规则、发布校验、运行机制和复用规范，作为 PRD、系统架构设计和数据库设计之间的中间约束层。

---

## 二、节点标准模型

每个节点由以下部分组成：

1. **节点元信息**
   - 节点 ID
   - 节点类型
   - 节点名称
   - 节点分类
   - 节点版本
   - 描述说明

2. **输入点**
   - 输入点 ID
   - 输入点名称
   - 变量名
   - 数据类型
   - 是否必填
   - 输入方式（仅连线 / 仅直接输入 / 连线或直接输入）
   - 是否允许动态新增

3. **执行配置**
   - 节点参数
   - 默认值
   - 校验规则
   - 渠道适配规则
   - 超时与异常处理策略

4. **输出点**
   - 输出点 ID
   - 输出点名称
   - 数据类型
   - 变量名
   - 输出语义说明

## 三、节点分类规范

### 3.1 获取内容类

- 全局变量
- 用户输入
- 对话记录获取

规范要求：
- 用户输入节点通常作为流程入口节点
- 对话记录获取节点应支持最近 N 条配置
- 全局变量节点支持多种输入控件和数据类型

### 3.2 输出类

- 消息输出
- 文件输出
- 视频输出
- 语音输出
- 图片输出
- 链接卡片输出
- Markdown 输出

规范要求：
- 输出节点必须显式声明发送内容类型
- 消息输出节点应支持分段策略配置
- 文件、图片、视频、语音既可引用素材库，也可引用外部 URL（视节点能力而定）

### 3.3 工具类

- 脚本运行
- HTTP 请求
- 设置客户属性
- 获取客户属性

规范要求：
- 脚本节点仅允许标准受控运行环境
- HTTP 节点需声明方法、地址、Headers、Query、Body 结构
- 客户属性作为会话持久状态，需与全局变量区分

### 3.4 流程控制类

- 取消 AI 托管
- 多重判断器
- 时间控制

规范要求：
- 判断器节点应支持布尔输出和分支路径控制
- 时间控制节点必须定义等待时长单位和上限
- 托管控制节点需明确生效时机（本轮结束后或即时）

### 3.5 逻辑处理类

- 正则节点
- JSON 节点
- AI 对话节点
- 知识库搜索
- 文本包装节点
- 高级知识库搜索

规范要求：
- AI 对话节点需定义模型、提示词、上下文输入来源
- JSON 节点解析失败时，应输出异常状态或中断后续执行
- 知识检索节点需声明知识库范围、检索模式和返回数量

### 3.6 复合类

- 智能体嵌入
- 子流程调用

规范要求：
- 复合节点需暴露标准输入输出契约
- 子流程更新后，引用方需明确刷新/重新拉取机制

### 3.7 特殊渠道类

- 获取企业微信标签
- 设置企业微信标签
- 拉群
- 获取 Morphix 标签
- 设置 Morphix 标签

规范要求：
- 节点必须声明适用渠道范围
- 对授权、托管状态、发布状态存在前置依赖时，需在节点配置侧明确提示

## 四、数据类型规范

系统统一支持以下数据类型：

| 类型 | 说明 |
|------|------|
| string | 文本 |
| text | 长文本 / 段落 |
| boolean | 布尔值 |
| number | 数值 |
| json | JSON 字符串或对象 |
| array | 列表 |
| message | 标准消息对象 |
| image | 图片 |
| audio | 音频 |
| video | 视频 |
| file | 文件 |
| markdown | Markdown 文本 |
| any | 任意类型 |
| knowledge | 检索到的知识结果 |
| customer_property | 客户属性值 |

规则：
1. 同类型输入输出点可直接连线。
2. `any` 类型输入点可接受任意输出类型。
3. 类型不匹配时，画布层禁止连线或给出明确警告。

## 五、变量规范

### 5.1 全局变量

- 使用范围：当前编排任意节点
- 引用方式：`{变量名}`
- 来源：
  - 系统默认变量
  - 全局变量节点自定义变量
  - 运行时注入变量

默认示例：
- `{userChatInput}`
- `{AIAnalyzeChatInput}`
- `{nowTime}`

### 5.2 局部变量

- 使用范围：仅当前节点配置内部
- 来源：节点输入点
- 引用方式：`{变量名}`

### 5.3 客户属性变量

- 使用范围：跨轮会话持久存在
- 通过“设置客户属性 / 获取客户属性”节点读写
- 不等同于全局变量；生命周期更长、作用域更偏向客户实体

### 5.4 变量命名规范

1. 采用英文或驼峰命名，如 `userIntent`、`chatHistory`
2. 禁止与系统默认变量重名
3. 节点新增输入点时必须指定唯一变量名

## 六、连线规范

### 6.1 连线合法性

仅满足以下条件时允许连线：

1. 输出点与输入点数据类型兼容
2. 不形成非法环路（除系统明确支持的循环机制）
3. 目标输入点未达到单输入上限
4. 节点间不存在被平台禁止的跨域引用

### 6.2 连线约束

1. 单个输入点是否允许多个来源，由节点定义决定。
2. 必填输入点未赋值时，不得保存。
3. 被删除节点的关联边需自动清理。
4. 复制节点时，默认不复制外部连线，避免形成错误依赖。

## 七、发布校验规范

保存或发布前必须通过以下校验：

### 7.1 结构校验

- 必填输入点全部有值
- 所有节点参数格式正确
- 所有连线类型合法
- 入口节点至少存在一个
- 输出链路满足最小可执行闭环要求

### 7.2 语义校验

- AI 节点提示词中引用的变量必须存在
- 知识库节点绑定的知识库必须可访问
- 渠道节点的渠道依赖必须满足
- 子流程节点引用的子流程版本必须有效

### 7.3 治理校验

- 编辑者是否有保存权限
- 发布者是否有发布权限
- 版本备注是否已填写
- 发布目标环境是否可用

---

## 八、运行机制规范

### 8.1 触发机制

1. 无输入点节点在流程启动时可直接运行。
2. 有输入点节点在所有必需输入满足后进入可运行状态。
3. 同时满足条件的多个节点，按系统调度策略排队执行。

### 8.2 执行顺序

- 默认按数据依赖驱动执行，不按画布位置决定。
- 同层可并发节点允许并发执行，但需记录实际顺序和开始结束时间。

### 8.3 异常处理

节点异常分为三类：

1. **阻断型**：错误发生后停止后续链路
2. **跳过型**：记录错误并跳过当前节点后继续
3. **降级型**：使用默认值或兜底逻辑继续执行

### 8.4 调试记录

每次运行至少记录：
- 会话 ID
- 工作流版本
- 节点执行顺序
- 节点输入值
- 节点输出值
- 执行耗时
- 异常信息
- 最终输出

## 九、复用规范

### 9.1 子流程规范

1. 子流程必须定义清晰的输入输出接口。
2. 子流程应支持独立版本管理。
3. 子流程更新后，主流程可选择：
   - 自动跟随最新版
   - 固定引用某个版本

### 9.2 模板规范

1. 流程模板可在项目内复用。
2. 跨项目复制时需处理：
   - 知识库映射
   - 素材库映射
   - 渠道节点适配差异
   - 权限与发布目标映射

### 9.3 智能体嵌入规范

1. 嵌入节点调用的目标机器人必须已发布。
2. 嵌入的输入输出需符合标准契约。
3. 被嵌入机器人升级后，调用方需评估兼容性。

---

## 十、节点能力扩展规范

新增节点时必须补齐：

1. 节点元信息
2. 输入输出定义
3. 参数面板配置
4. 数据类型说明
5. 校验规则
6. 运行时执行器实现说明
7. 调试展示结构
8. 权限与渠道依赖说明

## 十一、建议的节点协议字段

```yaml
NodeDefinition:
  type: string
  category: string
  name: string
  version: string
  description: string
  inputs:
    - id: string
      name: string
      varName: string
      dataType: string
      required: boolean
      inputMode: connect_only | input_only | both
      allowMultipleSources: boolean
  outputs:
    - id: string
      name: string
      varName: string
      dataType: string
  configSchema:
    - key: string
      label: string
      fieldType: string
      required: boolean
      defaultValue: any
  runtime:
    executorType: builtin | script | adapter
    timeoutMs: number
    errorPolicy: block | skip | fallback
```
