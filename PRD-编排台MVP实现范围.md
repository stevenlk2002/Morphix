# 编排台 MVP 实现范围确认

> **版本**：v1.0 | **日期**：2026-07-11 | **作者**：许清楚（PM）
> **前置依赖**：《PRD-无代码编排工作流.md》v1.0、《节点与流程规范-无代码编排工作流.md》v1.0、prototype/index.html

---

## 一、本次 MVP 目标

**在 morphix-console 前端内，基于 React Flow 实现可视化工作流编排台，用户可以拖拽节点、连线、配置参数并保存工作流至 localStorage，验证编排交互的完整闭环。**

---

## 二、实现范围

### P0 — 本次必做

| 编号 | 功能 | 说明 |
|------|------|------|
| P0-01 | 路由接入 | `src/router.tsx` 新增 `/bots/:botId/orchestrate` → `OrchestratePage` |
| P0-02 | 编排台布局 | Header（bot 名称 + 最后编辑时间 + 保存按钮）+ 左侧节点面板 + 中间画布 + 右侧属性面板 |
| P0-03 | 节点面板（3 Tab） | Tab 分类：基础节点 / 复合节点 / 特殊渠道节点；顶部搜索框实时过滤 |
| P0-04 | 节点拖入画布 | 从面板拖拽节点到画布，生成节点卡片（含节点图标、名称、输入/输出端口） |
| P0-05 | 7 种端口类型 + 颜色 | string(蓝) / number(橙) / boolean(绿) / chatHistory(青) / knowledgeRef(红) / any(黄) / property(紫)；仅同类型或 any 可连线 |
| P0-06 | SVG 贝塞尔连线 | 从输出端口拖拽到输入端口建立连线；不合法的连线目标视觉禁止 |
| P0-07 | 画布交互 | 拖动画布、滚轮缩放、节点选中高亮、Delete 键删除节点/连线 |
| P0-08 | 属性面板 | 点击节点后右侧展示该节点的 configSchema 表单（text/textarea/select/number/note 控件） |
| P0-09 | 默认工作流 | 页面首次加载展示 2 个占位节点：智能体嵌入 + 强提醒子流程（无连线） |
| P0-10 | 保存到 localStorage | 工作流数据（nodes + edges + botId）序列化存入 localStorage；刷新页可恢复 |
| P0-11 | 保存前校验 | 必填输入点（required: true）未赋值时弹 toast 提示具体节点和字段 |

### P1 — 建议本次做

| 编号 | 功能 | 说明 |
|------|------|------|
| P1-01 | 变量引用语法高亮 | config 文本框中 `{变量名}` 以蓝色高亮显示，提示用户它是变量 |
| P1-02 | 节点复制/粘贴 | Ctrl+C/V 复制选中节点（新节点 ID 重新生成，连线不复制） |
| P1-03 | 画布缩略图 | 右下角 mini-map 辅助大画布导航 |
| P1-04 | 连线悬停高亮 | hover 连线时加粗并显示删除按钮 |
| P1-05 | 工作流导出 JSON | "导出"按钮将当前工作流序列化为 JSON 文件下载 |
| P1-06 | 空画布引导 | 无节点时显示 "拖拽左侧节点到此处开始编排" 占位提示 |

### 本次不做

- 后端 API 对接（全量 mock-first，localStorage 为数据源）
- 发布/上线/回滚/版本管理
- 对话测试与逐节点调试
- RBAC 权限控制、审计日志
- 知识库/素材库管理
- 子流程打包/模板复用
- 多人协同编辑
- 节点级运行/执行引擎（编排台前端仅为编辑态）
- 多项目隔离

## 三、节点实现范围

### 本次实现 13 个节点（覆盖核心链路）

| 分类 | 节点 | 原型 id | 理由 |
|------|------|---------|------|
| 获取内容 | **用户输入** | userInput | 流程入口，无输入点 |
| 获取内容 | **全局变量** | globalVar | 变量定义能力 |
| 获取内容 | **对话记录获取** | chatHistory | 多轮记忆链路 |
| 逻辑处理 | **AI 对话** | aiChat | 核心大模型调用节点 |
| 逻辑处理 | **知识库搜索** | kbSearch | 知识检索注入 AI |
| 逻辑处理 | **正则提取** | regexExtract | 结构化提取 |
| 逻辑处理 | **JSON 提取** | jsonExtract | 结构化提取 |
| 输出 | **消息输出** | msgOutput | 最常用输出节点 |
| 输出 | **图片输出** | imageOutput | 多模态输出代表 |
| 流程控制 | **多重判断器** | multiJudge | 分支逻辑 |
| 流程控制 | **时间控制** | timeControl | 延时控制 |
| 工具 | **设置客户属性** | setCustomerAttr | 客户状态持久化 |
| 复合 | **智能体嵌入** | agentEmbed | 默认工作流节点，子流程入口 |

### 占位展示（不可配置，仅显示卡片）

| 节点 | 说明 |
|------|------|
| 强提醒子流程 | 默认工作流第二个节点，展示子流程调用占位形态 |

### 本次不做的节点

其余 31 个节点（文件/视频/语音/链接卡片/Markdown/邮件/小程序输出、脚本运行、HTTP 请求、获取客户属性、取消 AI 托管、智能知识库搜索、高级知识库搜索、文本包装/转义、FAQ/纠偏/知识中心搜索、11 个其余子流程、企微标签/Morphix 标签等特殊渠道节点）均留到后续迭代。

## 四、技术约定

| 项 | 约定 |
|----|------|
| 流程图引擎 | React Flow（`@xyflow/react`），已有原型验证 |
| 数据持久化 | `localStorage`，key 格式 `morphix_workflow_{botId}` |
| 组件库 | MUI + Tailwind CSS（与现有 morphix-console 保持一致） |
| 路由 | `/bots/:botId/orchestrate`，从 BotDetailPage 增加入口按钮 |
| mock 策略 | 知识库列表、AI 模型列表、素材列表在下拉框中用静态 mock 数据 |
| 节点 schema | 使用原型中已定义的 `NODE_SCHEMAS` 结构（inputs/outputs/config） |

## 五、待确认问题

1. **编排台入口**：从 Bot 详情页的哪个位置进入编排台？是顶部 Tab 切换（"配置 / 编排 / 日志"）还是独立按钮？建议 Tab 切换，因编排是 Bot 核心配置之一。

2. **智能体嵌入节点范围**：MVP 中智能体嵌入节点是否需要真实拉取已有 Bot 列表，还是先用 mock 下拉项展示？建议 mock 3-5 个假 bot 名即可，真实数据等后端 API 完成后再接。

3. **保存按钮行为**：保存时是否弹出 "版本备注" 输入框（为后续版本管理预留），还是直接静默保存？建议 MVP 直接静默保存 + toast 提示"已保存"，版本管理留到后续。
