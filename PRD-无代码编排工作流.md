# 无代码编排工作流产品需求文档（PRD）

> **版本**：v1.0 | **日期**：2026-07-11 | **状态**：初稿
> **产品定位**：面向多项目、多渠道智能运营平台的可视化无代码编排子系统

---

## 一、产品概述

### 1.1 产品定位

无代码编排工作流子系统用于帮助运营、产品、实施和 AI 配置人员，以拖拽节点、连接数据流、配置提示词与规则的方式，快速搭建和迭代机器人工作流，而无需直接编写后端代码。

该子系统作为多项目私域运营 AI 协同平台的核心能力之一，负责承接“用户输入 -> 意图理解 -> 知识检索 -> 大模型生成 -> 渠道动作 -> 标签沉淀 -> 运行监控”的流程编排。

### 1.2 解决的问题

当前机器人能力搭建往往存在以下问题：

- 依赖研发改逻辑，迭代周期长
- 机器人流程不可视、不可调试、不可复用
- 知识库、素材库、变量、渠道动作相互割裂
- 一旦涉及多渠道、多项目、多知识域，配置复杂度迅速上升
- 缺少发布、回滚、权限、审计等治理能力，难以进入正式运营阶段

本系统通过可视化编排和标准化节点体系，将复杂的对话和自动化能力沉淀为业务可配置资产。

### 1.3 产品目标

| 目标 | 说明 |
|------|------|
| 提升搭建效率 | 将机器人流程搭建周期从“天级/周级”缩短到“小时级” |
| 降低研发依赖 | 让产品、运营、实施在大部分场景下可独立完成流程配置 |
| 支持多渠道扩展 | 在统一编排模型下适配企微、微信个人号、WhatsApp 等渠道 |
| 支持多项目隔离 | 不同项目拥有独立的流程、知识库、素材和发布版本 |
| 建立治理闭环 | 支持调试、发布、回滚、权限控制、审计追踪 |

### 1.4 目标用户

- **项目负责人**：定义业务目标、审批发布、管理权限
- **运营配置人员**：搭建和调整机器人工作流
- **AI/Prompt 配置人员**：优化 AI 对话节点、知识调用和多轮回复策略
- **实施顾问**：按行业/客户需求快速复制与定制流程
- **测试/质控人员**：执行对话调试、回归验证、版本验收

### 1.5 核心价值

- 可视化表达复杂流程，降低理解和维护成本
- 节点化沉淀能力，促进模板复用和子流程复用
- 统一接入知识库、素材库、变量、脚本、HTTP、渠道动作
- 建立运行时可观测能力，支持逐节点调试与问题排查
- 将工作流纳入版本发布和审计治理，支持正式生产使用

## 二、范围定义

### 2.1 首版范围（In Scope）

1. 可视化画布编排
2. 节点拖拽、连线、校验、保存
3. 节点分类体系与节点配置
4. 全局变量、局部变量、客户属性
5. AI 对话、知识库搜索、高级知识库搜索
6. 素材输出、Markdown 输出、链接卡片输出等多种输出方式
7. 子流程调用、智能体嵌入
8. 多渠道特殊动作节点（以企微/Morphix 为首批）
9. 对话测试与逐节点调试明细
10. 发布、上线、回滚、版本管理
11. 角色权限、操作审计、运行日志
12. 多项目隔离与模板复用机制

### 2.2 暂不纳入首版（Out of Scope）

1. 图形化拖拽连线的多人实时协同编辑
2. 自动生成完整流程图的 AI Copilot 编排助手
3. 节点级计费结算系统
4. 第三方市场化节点插件生态

## 三、核心概念

### 3.1 编排核心实体

| 概念 | 说明 |
|------|------|
| 项目 Project | 多项目平台中的业务隔离单元 |
| 机器人 Bot | 面向具体业务场景的对话或流程执行主体 |
| 工作流 Workflow | 由节点和边组成的流程定义 |
| 节点 Node | 最小功能执行单元 |
| 连线 Edge | 节点之间的数据传递关系 |
| 输入点 / 输出点 | 节点接收和输出数据的接口 |
| 局部变量 | 节点输入点对应的局部可用变量 |
| 全局变量 | 当前编排内任意节点可引用的变量 |
| 客户属性 | 面向客户会话长期保存的状态变量 |
| 子流程 | 可被其他工作流复用的节点组合 |
| 发布版本 | 可上线运行的工作流快照 |
| 调试会话 | 对话测试中的一次运行记录 |

### 3.2 节点分类

| 分类 | 典型节点 |
|------|---------|
| 获取内容类 | 用户输入、对话记录获取、全局变量 |
| 输出类 | 消息输出、文件输出、图片输出、语音输出、视频输出、链接卡片输出、Markdown 输出 |
| 工具类 | 脚本运行、HTTP 请求、设置客户属性、获取客户属性 |
| 流程控制类 | 多重判断器、时间控制、取消 AI 托管 |
| 逻辑处理类 | 正则节点、JSON 节点、AI 对话节点、知识库搜索、文本包装、高级知识库搜索 |
| 复合类 | 智能体嵌入、子流程调用 |
| 特殊渠道类 | 获取/设置企业微信标签、获取/设置 Morphix 标签、拉群 |

## 四、业务场景

### 4.1 快速构建基础机器人

用户通过“用户输入 -> AI 对话 -> 消息输出”搭建基础问答机器人，实现最小可运行闭环。

### 4.2 构建有记忆的多轮机器人

通过“对话记录获取”节点将最近 N 条历史消息传递给 AI 对话节点，让机器人具备短期上下文记忆。

### 4.3 构建专业知识机器人

通过“知识库搜索 / 高级知识库搜索”节点，把知识结果注入 AI 对话节点，形成面向单领域或多领域的专业助手。

### 4.4 构建多分支决策流程

通过“多重判断器、正则节点、JSON 节点、时间控制”等节点，实现按意图、关键词、结构化字段和延时规则的分支执行。

### 4.5 构建渠道动作机器人

在对话过程中根据条件打标签、获取标签、拉群、切换托管状态，完成与渠道运营动作的联动。

### 4.6 构建复用型流程资产

将成熟流程打包为子流程或嵌入已有机器人，支持跨业务场景复用，提高交付效率。

## 六、业务规则与约束

### 6.1 保存校验规则

1. 只有当所有带 `*` 的输入点均已赋值时，整个编排才允许保存。
2. 赋值方式包括：
   - 连线传值
   - 输入框直接输入
3. 若任一必填输入点缺失，需明确提示未通过校验的节点和字段。

### 6.2 类型匹配规则

1. 只有输出点和输入点类型相同，才允许连线。
2. 若输入点为“任意类型”，则允许连接任意输出类型。
3. 新增输入点时需同时定义变量名和数据类型。

### 6.3 运行规则

1. 无输入点节点，在流程启动时可直接运行。
2. 有输入点节点，只有在其左侧被连接输入点均收到值后才可运行。
3. 多个节点同时满足运行条件时，需要进入排队执行。
4. 某节点执行失败时，应根据节点类型决定：阻断后续、跳过后续、写入异常日志。

### 6.4 变量规则

1. 全局变量和局部变量均通过 `{变量名}` 引用。
2. 局部变量仅能在当前节点配置中引用。
3. 全局变量可在当前编排任意节点中使用。
4. 脚本运行节点中的文本变量引用，需要自动补齐安全转义和引号规则。

### 6.5 知识检索规则

1. 语义搜索：仅检索“搜索内容”字段。
2. 全文搜索 / 混合搜索：同时检索“搜索内容”和“辅助内容”。
3. 高级知识库需预先配置字段类型与是否支持向量检索。
4. 知识库搜索节点支持同时选择多个知识库。

### 6.6 渠道限制规则

1. 企业微信标签节点依赖代开发授权。
2. Morphix 标签能力仅在托管状态下可用。
3. 编排更新后，已托管机器人需重新上线后才能对用户生效。
4. 设置企业微信标签后，当前轮流程内不保证立即可读，通常下一轮对话可见。

## 八、非功能需求

### 8.1 性能

| 指标 | 目标 |
|------|------|
| 画布加载时间 | 常规 200 节点工作流小于 3 秒 |
| 发布耗时 | 单工作流发布小于 10 秒 |
| 节点调试明细查询 | 单次调试记录打开小于 2 秒 |
| 知识检索延迟 | P99 小于 500ms |
| 对话响应链路 | P99 小于 3 秒（不含外部模型极端抖动） |

### 8.2 可用性

| 指标 | 目标 |
|------|------|
| 系统可用性 | ≥ 99.9% |
| 历史版本可恢复性 | 任一已发布版本可回滚 |
| 调试记录保留 | 至少保留 90 天 |
| 审计日志保留 | 至少保留 180 天 |

### 8.3 安全与合规

| 要求 | 说明 |
|------|------|
| RBAC 权限控制 | 关键动作需校验项目角色和操作权限 |
| 审计留痕 | 编辑、发布、回滚、授权必须可追踪 |
| 敏感信息保护 | API Key、渠道凭证、私有知识库数据需加密存储 |
| 数据隔离 | 项目之间工作流、知识库、素材库、调试数据严格隔离 |

### 8.4 扩展性

| 维度 | 策略 |
|------|------|
| 节点扩展 | 节点协议标准化，支持后续新增节点类型 |
| 渠道扩展 | 渠道动作走适配器模式 |
| 知识扩展 | 支持普通知识库和高级知识库统一抽象 |
| 模板扩展 | 支持子流程、模板、机器人嵌入复用 |

## 九、验收标准

| 编号 | 验收项 | 通过标准 |
|------|--------|---------|
| F-01 | 基础流程搭建 | 用户可通过用户输入、AI 对话、消息输出搭建可运行机器人 |
| F-02 | 保存校验 | 缺失必填输入点时不可保存，并能定位错误节点 |
| F-03 | 类型校验 | 不同类型输入输出点不可错误连线 |
| F-04 | 变量引用 | 全局变量和局部变量可正确在节点中引用 |
| F-05 | 对话记忆 | 对话记录获取节点能为 AI 节点提供最近 N 条历史记录 |
| F-06 | 知识检索 | 机器人可基于普通知识库/高级知识库稳定返回检索结果 |
| F-07 | 子流程复用 | 已封装子流程可在新流程中被调用并成功运行 |
| F-08 | 调试可观测 | 对话测试中可查看逐节点输入/输出/顺序/异常 |
| F-09 | 发布回滚 | 工作流支持发布新版本和回滚旧版本 |
| F-10 | 权限审计 | 未授权角色不能发布，关键操作可在审计日志中追踪 |

## 十、术语表

| 术语 | 说明 |
|------|------|
| Workflow | 工作流，由节点和边构成的可执行流程 |
| Node | 节点，工作流中的最小能力单元 |
| Edge | 连线，节点间的数据传递关系 |
| Global Variable | 全局变量，可在当前编排任意节点引用 |
| Local Variable | 局部变量，只能在当前节点配置中使用 |
| Debug Session | 调试会话，一次测试运行的完整执行记录 |
| Publish Version | 发布版本，可上线运行的工作流快照 |
| Subflow | 子流程，可复用的节点组合 |
| Channel Adapter | 渠道适配器，将流程动作转为具体渠道执行命令 |
